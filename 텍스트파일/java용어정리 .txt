요리 재료 (매개 변수): 요리 재료는 요리를 만들기 위한 "실제 재료"입니다. 예를 들어, 요리할 때 사용할 "양파", "감자" 같은 것이 재료입니다.
매개 변수는 요리법에서 사용되는 재료라고 할 수 있습니다. 즉, 어떤 "구체적인 값"을 나타냅니다.
요리법 (참조 변수): 요리법은 재료를 어떻게 사용할지 알려주는 "설명서"라고 할 수 있습니다. 요리법에서 실제 재료를 사용할 수 있도록, 재료들이 어떤 "그릇"에 담겨있는지 가리킵니다.
참조 변수는 "그릇"처럼 객체가 어디에 저장되어 있는지 알려주는 역할을 합니다. 참조 변수는 "주소"를 가지고 있어, 해당 객체를 찾아서 그 객체에 접근할 수 있게 해줍니다.





sort와 compareTo는 자바가 기본적으로 제공하는 메서드
sort는 리스트나 배열을 정렬하는 데 사용되는 메서드입니다. 자바에서 List 인터페이스는 sort 메서드를 제공하여, 리스트의 요소들을 오름차순 또는 내림차순으로 정렬할 수 있게 해줍니다

compareTo는 두 객체를 비교하는 메서드입니다. compareTo는 Comparable 인터페이스에 정의되어 있고, 주로 두 문자열이나 숫자를 비교할 때 사용됩니다.

1. 매개변수 (Parameter)
매개변수는 어떤 함수나 메서드를 호출할 때 필요한 정보입니다. 쉽게 말해서, 함수에게 전달하는 값입니다.


 public int add(int num1, int num2) {
        return num1 + num2;

add(int num1, int num2)는 두 개의 숫자를 더하는 함수입니다. num1과 num2는 이 함수가 실행되려면 반드시 필요로 하는 매개변수입니다.


비유:
요리 레시피를 생각해 보세요.
요리 레시피에서 **"재료"**는 그 요리를 만들 때 필요한 것들이죠. 이 재료들이 바로 매개변수입니다.
예를 들어, "계란"과 "밀가루"라는 재료가 필요한 오믈렛을 만드는 레시피가 있다면, "계란"과 "밀가루"가 매개변수입니다.




인터페이스와 클래스의 차이점
인터페이스 (Interface): [interface Animal]

규칙/약속을 정의합니다.   
클래스가 이 규칙을 따라서 구현해야 합니다.
클래스가 인터페이스를 구현(implement)하여, 그 인터페이스에서 정의한 기능을 실제로 동작하도록 만듭니다.


클래스 (Class):  [class Dog implements Animal]

구체적인 구현을 정의합니다.
객체 지향 프로그래밍에서 실제 동작을 하는 구체적인 코드를 포함하고 있습니다.
인터페이스를 구현하거나, 상속해서 여러 기능을 구체적으로 실현합니다.





List는 인터페이스: 규칙을 정의하고, 구체적인 동작은 구현체에서 정의합니다.
ArrayList는 클래스: List 인터페이스를 구현한 배열 기반의 동적 리스트입니다.
비유를 통한 설명:
**List**는 **"리스트가 어떻게 동작해야 하는지에 대한 계약서"**입니다. 이 계약서에는 "요소를 추가하고, 제거하고, 조회하는 방식"에 대한 규칙만 있습니다.
**ArrayList**는 그 계약서를 실제로 실행하는 직원입니다. 이 직원은 계약서에 있는 규칙을 배열을 사용해 실제로 구현한 방식입니다.
